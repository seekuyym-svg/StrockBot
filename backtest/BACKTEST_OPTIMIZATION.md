## 一、当前系统架构

```
选股 (generate_stockpool.py)   → 7大条件筛选
    ↓
评分 (score_stockpool.py)      → 100分制 + 优选分(0~10)
    ↓
回测 (run_backtest.py + backtest_engine.py)
    ↓
信号 (trade_decision/market_signal.py)  → L1大盘环境过滤
```

## 二、选股条件（最终版）

### 条件①：成交量放量

| 子条件 | 内容 | 配置参数 |
|:-----:|:------|:---------|
| **1b** | 至少2天 ≥ 基准日×85% | `volume_up_ratio: 0.85` |
| **底线** | 所有天 ≥ 基准日×65% | `volume_floor_ratio: 0.65` |
| **1c** | 第3天不是最小的，且 ≥ 基准日 | — |

### 条件②：价格稳步上涨

| 子条件 | 内容 |
|:-----:|:------|
| **2a** | 至少2天收盘 > 基准日收盘 |
| **2b** | 低于基准日的天 ≥ 基准日收盘×95% |
| **2c** | 第3天不是3天中最低的 |
| **2d** | 每天冲高回落 < 8% |

### 条件③~⑦

| # | 条件 | 内容 | 配置参数 |
|:-:|:----|:-----|:---------|
| **③** | 站上MA20 | 收盘 > 20日均线 | — |
| **④** | 涨幅区间 | 放量期涨幅 ∈ [0%, 20%] | `min/max_price_change_pct` |
| **⑤** | 量比区间 | 放量期3天 ≥ 1.0x，基准日 ≥ 0.9x | `min_volume_ratio`, `base_volume_ratio`, `max_volume_ratio` |
| **⑥** | 相对强度 | 股票涨幅 - 沪深300涨幅 ≥ -2% | `min_relative_strength` |
| **⑦** | 波动率过滤 | 日收益率标准差 < 6.0% | `max_volatility_pct` |

## 三、评分工具优化

### 评分维度（100分制）

| 维度 | 权重 | 说明 |
|:----:|:----:|:------|
| 量能因子 | 20分 | 成交量递增、量比、换手率 |
| 趋势因子 | 35分 | 均线多头排列、均线斜率 |
| 动量因子 | 20分 | 5日/10日涨幅、相对强度 |
| 形态因子 | 15分 | MACD、布林带位置 |
| 风险因子 | 10分 | RSI、波动率控制 |

### 优选分（0~10分）

评分相同时的二次排序依据：

| 维度 | 满分 | 条件 |
|:----:|:----:|:------|
| ①成交量和放量 | 3分 | 3天量递增+量比<1.3x |
| ②第3天最高 | 2分 | 收盘价是3天最高 |
| ③涨幅<10% | 3分 | 涨幅在10%以内 |
| ④冲高回落<3% | 2分 | 上影线小于3% |

### 评分选择流程

```
候选股评分60~100
  ↓ 评分区间筛选 (min_score ~ max_score)
  ↓ 按 (评分降序, 优选分降序) 排序
  ↓ 分层采样（等距取20只，每个分值最多3只）
  ↓ 行业分散（每行业最多3只）
  ↓ 最终10只
```

## 五、修复的Bug

| Bug | 原因 | 修复 |
|:----|:-----|:------|
| K线数据获取日期截断顺序错误 | 先截取最近N天，再按end_date过滤 → 0条 | 调换顺序：先过滤日期，再截取天数 |
| mootdx读取本地数据用带前缀代码 | `daily(symbol="sz300183")` mootdx不支持 | 改为纯数字代码 `daily(symbol="300183")` |
| 行业信息获取连接被重置 | akshare请求频率过高 | 三层数据源：东方财富HTTP→akshare→本地JSON缓存 |
| 评分工具本地数据读取失败 | 未复权问题 | mootdx std模式即前复权 ✅ |
| 回测5月底~6月初亏损 | 成交量放宽后假放量股票混入 | 收紧成交量条件（至少2天≥85%+底线65%） |

## 七、配置文件（config.yaml 关键参数）

  hold_days: 2                    # 持仓天数
  volume_period: 3                # 放量周期
  min_score: 65                   # 最低评分
  max_score: 75                   # 最高评分
  max_stocks_per_cycle: 10        # 每周期最多10只
  min_stocks_per_cycle: 3         # 最少3只
  max_stocks_per_industry: 3      # 每行业最多3只
  min_price_change_pct: 0.0
  max_price_change_pct: 20.0
  min_volume_ratio: 1.0
  max_volume_ratio: 5.0
  base_volume_ratio: 0.9
  max_retracement_pct: -5.0
  max_intraday_pullback_pct: 8.0
  min_relative_strength: -2.0
  max_volatility_pct: 6.0
  volume_up_ratio: 0.85
  volume_floor_ratio: 0.65

  ma_short: 20
  ma_long: 60
  min_market_volume: 25000

breakout:
  consolidation_days: 60
  consolidation_range: 15.0
  vol_ratio_threshold: 1.5
  pullback_threshold: 5.0
