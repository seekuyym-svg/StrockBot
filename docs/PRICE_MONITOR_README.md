# 价格极值监控功能说明

## 📋 功能概述

在原有的T+0 ETF交易信号定时提醒基础上，新增了**价格极值追踪与反转提醒**功能。当系统处于WAIT（观望）状态时，会自动监控价格的极端波动，并在价格反转时通过飞书发送提醒通知。

---

## 🎯 核心逻辑

### 1️⃣ 卖出监控（高位回落提醒）

**触发条件**：
- 当日涨幅 ≥ **3.0%**（可配置）

**监控过程**：
- 持续追踪并更新最高价
- 计算从最高点回落的幅度

**提醒条件**：
- 从最高点回落 ≥ **0.5%**（可配置）
- 发送飞书通知，包含：历史最高价、当前价格、回落幅度

**示例**：
```
🔴 卖出提醒（价格监控）
标的: 港股创新药ETF (sh.513120)
类型: WAIT
价格: ¥1.245
涨跌: +2.85%
BOLL: 上轨-2.34% | 中轨+1.56% | 下轨+5.67%
RSI: 68.45 (🟡 中性区)
💡 研判: 暂无

原因: 价格监控：从最高点¥1.252回落0.56%
```

---

### 2️⃣ 买入监控（低位反弹提醒）

**触发条件**：
- 当日跌幅 ≥ **3.5%**（可配置）

**监控过程**：
- 持续追踪并更新最低价
- 计算从最低点反弹的幅度

**提醒条件**：
- 从最低点反弹 ≥ **0.5%**（可配置）
- 发送飞书通知，包含：历史最低价、当前价格、反弹幅度

**示例**：
```
🟢 买入提醒（价格监控）
标的: 中概互联网ETF (sh.513050)
类型: WAIT
价格: ¥0.985
涨跌: -3.12%
BOLL: 上轨-8.23% | 中轨-4.56% | 下轨-0.89%
RSI: 28.67 (🟢 超卖区 ✅)
💡 研判: 超卖反弹机会

原因: 价格监控：从最低价¥0.978反弹0.72%
```

## ⚙️ 配置说明

价格监控支持**全局默认配置**和**个性化配置**两种方式：

### 方式一：全局默认配置（适用于所有ETF）

在 `config.yaml` 中的 `scheduler.price_monitor` 节点配置全局默认值：

```yaml
scheduler:
  # ... 其他配置 ...
  
  # 价格极值追踪监控配置（仅在WAIT信号时生效，作为全局默认值）
  price_monitor:
    enabled: true  # 是否启用价格监控
    
    # 卖出监控参数
    sell_monitor:
      trigger_rise_pct: 3.0      # 触发监控的上涨幅度阈值（%）
      pullback_pct: 0.5          # 从最高点回落幅度阈值（%）
    
    # 买入监控参数
    buy_monitor:
      trigger_drop_pct: 3.5      # 触发监控的下跌幅度阈值（%）
      rebound_pct: 0.5           # 从最低点反弹幅度阈值（%）
```

### 方式二：个性化配置（推荐，每个ETF独立配置）

在每个 ETF 标的的配置中添加个性化价格监控参数：

```yaml
symbols:
  - code: "sh.513120"  # 港股创新药ETF
    name: "港股创新药ETF"
    enabled: true
    # ... 其他策略参数 ...
    
    # 个性化价格监控参数（可选，不设置则使用全局默认值）
    price_monitor_enabled: true  # 是否启用价格监控
    sell_trigger_rise_pct: 3.0   # 卖出监控触发涨幅阈值（%）
    sell_pullback_pct: 0.5       # 卖出监控回落幅度阈值（%）
    buy_trigger_drop_pct: 3.5    # 买入监控触发跌幅阈值（%）
    buy_rebound_pct: 0.5         # 买入监控反弹幅度阈值（%）
    
  - code: "sh.513050"  # 中概互联网ETF
    name: "中概互联网ETF"
    enabled: true
    # ... 其他策略参数 ...
    
    # 个性化价格监控参数（更激进的配置）
    price_monitor_enabled: true
    sell_trigger_rise_pct: 2.5   # 更早触发卖出监控
    sell_pullback_pct: 0.6       # 需要更大的回落才提醒
    buy_trigger_drop_pct: 3.0    # 更早触发买入监控
    buy_rebound_pct: 0.4         # 更小的反弹就提醒
```

### 配置优先级规则

1. **个性化配置优先**：如果 symbol 配置了价格监控参数，使用该值
2. **全局配置兜底**：如果 symbol 未配置某参数，使用 `scheduler.price_monitor` 的全局默认值
3. **向后兼容**：不配置个性化参数时，系统行为与之前完全一致

### 可配置的个性化参数

| 参数名 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `price_monitor_enabled` | bool | 是否启用价格监控 | 跟随全局配置 |
| `sell_trigger_rise_pct` | float | 卖出监控触发涨幅阈值（%） | 3.0 |
| `sell_pullback_pct` | float | 卖出监控回落幅度阈值（%） | 0.5 |
| `buy_trigger_drop_pct` | float | 买入监控触发跌幅阈值（%） | 3.5 |
| `buy_rebound_pct` | float | 买入监控反弹幅度阈值（%） | 0.5 |

### 参数调整建议

| 场景 | sell_trigger_rise_pct | sell_pullback_pct | buy_trigger_drop_pct | buy_rebound_pct |
|------|----------------------|-------------------|---------------------|-----------------|
| **保守型**（减少误报） | 5.0% | 1.0% | 5.0% | 1.0% |
| **平衡型**（默认） | 3.0% | 0.5% | 3.5% | 0.5% |
| **激进型**（更敏感） | 2.0% | 0.3% | 2.5% | 0.3% |

### 实际应用示例

#### 示例1：不同波动特性的ETF使用不同配置

```
symbols:
  # 波动较小的ETF - 使用较敏感的监控
  - code: "sh.513120"
    name: "港股创新药ETF"
    price_monitor_enabled: true
    sell_trigger_rise_pct: 2.5   # 较小涨幅就监控
    sell_pullback_pct: 0.4       # 小幅回落就提醒
    buy_trigger_drop_pct: 2.8    # 较小跌幅就监控
    buy_rebound_pct: 0.4         # 小幅反弹就提醒
  
  # 波动较大的ETF - 使用较保守的监控
  - code: "sh.513050"
    name: "中概互联网ETF"
    price_monitor_enabled: true
    sell_trigger_rise_pct: 4.0   # 较大涨幅才监控
    sell_pullback_pct: 0.8       # 大幅回落才提醒
    buy_trigger_drop_pct: 4.5    # 较大跌幅才监控
    buy_rebound_pct: 0.8         # 大幅反弹才提醒
```

#### 示例2：禁用某个ETF的价格监控

```
symbols:
  - code: "sh.513120"
    name: "港股创新药ETF"
    price_monitor_enabled: false  # 禁用该ETF的价格监控
```

## 🔧 技术实现细节

### 1. 状态管理

每个ETF标的独立维护监控状态：

```python
{
    'sh.513120': {
        'sell': {
            'active': False,              # 是否激活监控
            'trigger_price': 0.0,         # 触发时的价格
            'highest_price': 0.0,         # 监控期间最高价
            'last_notify_time': 0         # 上次通知时间戳
        },
        'buy': {
            'active': False,
            'trigger_price': 0.0,
            'lowest_price': 0.0,
            'last_notify_time': 0
        }
    }
}
```

### 2. 频率控制

- **最小间隔**：同一标的同类型提醒最小间隔 **60秒**
- **日内单次**：一交易日内每个监控方向只触发一次，触发后自动重置

### 3. 数据持久化

- **无需持久化**：监控状态存储在内存中，重启后丢失
- **设计理由**：主要监控日内的涨跌幅（相较于昨日收盘价），一般当日交易时间内不会重启

```

  ↓

```

## ⚠️ 注意事项

1. **仅针对WAIT信号**：价格监控只在系统处于观望状态时生效，不影响正常的买卖信号逻辑
2. **基于昨日收盘价**：涨跌幅是相对于昨日收盘价计算的，不是相对于开盘价
3. **跨交易日重置**：每日开盘时监控状态会自动重置（因为是基于内存存储）
4. **频率限制保护**：避免短时间内重复发送相同类型的通知
5. **配置灵活性**：所有阈值均可通过配置文件动态调整，无需修改代码

## 📝 常见问题

### Q1: 为什么选择内存存储而不是数据库？
**A**: 价格监控主要针对日内交易，重启概率极低。使用内存存储可以简化实现，提高性能，且符合"日内单次触发"的设计要求。

### Q2: 如果上午触发了卖出监控，下午还能再次触发吗？
**A**: 不能。根据设计规范，一交易日内每个监控方向只触发一次。如果需要重新监控，需要等到下一个交易日。

### Q3: 如何临时禁用某个标的的价格监控？
**A**: 在该标的的配置中设置 `price_monitor_enabled: false` 即可。例如：
```
symbols:
  - code: "sh.513120"
    name: "港股创新药ETF"
    price_monitor_enabled: false  # 禁用该ETF的价格监控
```

如果需要全局禁用所有标的，可在配置文件中设置 `scheduler.price_monitor.enabled: false`。

### Q4: 飞书通知会打扰我吗？
**A**: 不会。系统设置了60秒的最小间隔，且一交易日内每个方向只触发一次，确保通知频率合理。
