# 买入委托定时任务调度器使用说明

## 📋 功能概述

该调度器实现了自动化的买入委托计算和通知功能：

1. **定时执行**：每周一至周五上午 9:26 自动触发（可配置）
2. **智能检查**：自动检查是否存在前一日的选股结果文件
3. **自动计算**：读取选股结果，过滤评分 >= 1.0 的股票，计算买入股数
4. **飞书通知**：将委托明细通过飞书推送给你

## 🚀 使用方法

## ⚙️ 配置说明

在 `config.yaml` 中配置买入委托调度器：

```yaml
# 买入委托定时任务配置
buy_order_scheduler:
  # 是否启用买入委托定时任务
  enabled: true
  # 执行时间（24小时制）
  hour: 9
  minute: 26
  # 最低综合评分阈值
  min_score: 1.0
```

**配置项说明：**
- `enabled`: 是否启用该定时任务（true/false）
- `hour`: 执行的小时（0-23）
- `minute`: 执行的分钟（0-59）
- `min_score`: 选股评分最低要求，低于此分数的股票会被过滤

**常用时间示例：**
- 每天 9:30: `hour: 9, minute: 30`
- 工作日 14:00: `hour: 14, minute: 0`

## ⚠️ 注意事项

1. **确保选股结果文件存在**：
   - 调度器依赖 `select_stocks_volume.py` 生成的选股结果
   - 如果前一天没有生成文件，调度器会跳过执行

2. **确保飞书配置正确**：
   - 检查 `config.yaml` 中的 Webhook URL
   - 测试飞书通知是否正常

3. **系统时间准确性**：
   - 确保服务器/电脑的系统时间准确
   - 建议使用 NTP 时间同步服务

## 🛠️ 故障排查

### 问题1：调度器未执行

**检查：**
- 系统时间是否正确
- 是否为工作日（周一到周五）
- 查看控制台是否有错误日志
- 检查 `config.yaml` 中 `buy_order_scheduler.enabled` 是否为 `true`

### 问题2：飞书通知未发送

**检查：**
- `config.yaml` 中 `feishu.enabled` 是否为 `true`
- Webhook URL 是否正确
- 网络连接是否正常

### 问题3：找不到选股结果文件

**解决：**
- 确认已运行 `python local/select_stocks_volume.py`
- 检查 `data/` 目录下是否有 `stockpool_YYYYMMDD.txt` 文件
- 确认日期是否正确（调度器查找的是前一天的文件）
