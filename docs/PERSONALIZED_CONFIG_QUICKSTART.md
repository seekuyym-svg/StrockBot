# 个性化配置快速上手指南

## 🚀 5分钟快速开始

### 步骤 1: 了解当前配置

打开 `config.yaml`，你会看到类似这样的配置：

```yaml
symbols:
  - code: "sh.513120"  # 港股创新药ETF
    name: "港股创新药ETF"
    enabled: true
    add_drop_threshold: 3.0  # 下跌3%加仓
    take_profit_threshold: 2.0  # 盈利2%止盈
    
  - code: "sh.513050"  # 中概互联网ETF
    name: "中概互联网ETF"
    enabled: true
    add_drop_threshold: 3.5  # 下跌3.5%加仓
    take_profit_threshold: 2.5  # 盈利2.5%止盈
```

### 步骤 2: 理解参数含义

| 参数 | 含义 | 影响 |
|------|------|------|
| `add_drop_threshold` | 加仓跌幅阈值 | 值越小，加仓越频繁；值越大，加仓越保守 |
| `take_profit_threshold` | 止盈涨幅阈值 | 值越小，止盈越快；值越大，追求更高收益 |

### 步骤 3: 调整参数

#### 场景 A: 想让某个 ETF 更激进

```yaml
- code: "sh.513120"
  name: "港股创新药ETF"
  enabled: true
  add_drop_threshold: 2.0  # 从3.0改为2.0，更频繁加仓
  take_profit_threshold: 1.5  # 从2.0改为1.5，更快止盈
```

**效果**：
- 下跌 2% 就加仓（原来是 3%）
- 盈利 1.5% 就卖出（原来是 2%）
- 交易频率提高，适合波动大的 ETF

#### 场景 B: 想让某个 ETF 更保守

```yaml
- code: "sh.513050"
  name: "中概互联网ETF"
  enabled: true
  add_drop_threshold: 4.5  # 从3.5改为4.5，减少加仓频率
  take_profit_threshold: 3.0  # 从2.5改为3.0，追求更高收益
```

**效果**：
- 下跌 4.5% 才加仓（原来是 3.5%）
- 盈利 3% 才卖出（原来是 2.5%）
- 交易频率降低，适合稳健操作

#### 场景 C: 只调整一个参数

```yaml
- code: "sh.513120"
  name: "港股创新药ETF"
  enabled: true
  add_drop_threshold: 2.5  # 只调整加仓阈值
  # take_profit_threshold 不设置，使用全局默认值
```

**效果**：
- 该 ETF 使用个性化的加仓阈值（2.5%）
- 止盈阈值使用全局默认值（在 `strategy.take_profit_threshold` 中定义）

### 步骤 4: 验证配置

运行测试脚本：

```bash
python test_personalized_config.py
```

查看输出，确认配置已生效：


观察启动日志，确认每个 ETF 的配置：

## 💡 实用技巧

### 技巧 1: 根据历史波动性设置

查看 ETF 的历史波动情况：
- **高波动**（日波动 > 3%）：设置较小的 `add_drop_threshold`（2.0-2.5%）
- **中波动**（日波动 1-3%）：设置中等的 `add_drop_threshold`（3.0-3.5%）
- **低波动**（日波动 < 1%）：设置较大的 `add_drop_threshold`（4.0-5.0%）

### 技巧 2: 根据持仓时间偏好设置

- **短线交易**（持仓 1-3 天）：
  ```yaml
  add_drop_threshold: 2.0
  take_profit_threshold: 1.0
  ```

- **中线交易**（持仓 1-2 周）：
  ```yaml
  add_drop_threshold: 3.5
  take_profit_threshold: 2.5
  ```

- **长线交易**（持仓 1 个月以上）：
  ```yaml
  add_drop_threshold: 5.0
  take_profit_threshold: 4.0
  ```

### 技巧 3: 渐进式调整

不要一次性大幅调整参数，建议：

1. **第一周**：使用默认配置观察
2. **第二周**：小幅调整（±0.5%），观察效果
3. **第三周**：根据实际表现继续优化
4. **持续迭代**：每月回顾一次，逐步找到最佳参数

### 技巧 4: 不同市场环境的调整

- **牛市**：可以适当提高止盈阈值，让利润奔跑
  ```yaml
  take_profit_threshold: 3.0  # 追求更高收益
  ```

- **熊市**：应该降低加仓阈值，快速止盈
  ```yaml
  add_drop_threshold: 2.0  # 谨慎加仓
  take_profit_threshold: 1.5  # 快速锁定利润
  ```

- **震荡市**：保持中等参数，平衡风险与收益
  ```yaml
  add_drop_threshold: 3.0
  take_profit_threshold: 2.0
  ```

## ⚠️ 常见误区

### ❌ 误区 1: 参数越小越好

**错误想法**：加仓阈值设得越小，就能越多地捕捉机会

**实际情况**：
- 过小的加仓阈值会导致频繁加仓
- 可能在大跌前就把资金用完
- 增加交易成本

**建议**：根据 ETF 的实际波动性设置，不要盲目追求小数值

### ❌ 误区 2: 所有 ETF 用相同参数

**错误想法**：统一参数便于管理

**实际情况**：
- 不同 ETF 的波动特性差异很大
- 创新药 ETF 和中概互联 ETF 的波动性完全不同
- 统一参数无法充分发挥每个 ETF 的优势

**建议**：为每个 ETF 量身定制参数

### ❌ 误区 3: 参数设置后不再调整

**错误想法**：找到一组好参数就可以一直用

**实际情况**：
- 市场环境会变化
- ETF 的波动特性也会变化
- 需要定期回顾和调整

**建议**：每月检查一次参数效果，适时调整
