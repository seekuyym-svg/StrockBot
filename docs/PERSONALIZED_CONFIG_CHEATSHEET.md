# 个性化配置快速参考卡片

## 📌 核心概念

**个性化配置** = 为每个 ETF 单独设置策略参数，而不是所有 ETF 使用相同的参数。

## ⚙️ 可配置的参数

| 参数 | 含义 | 推荐范围 | 影响 |
|------|------|----------|------|
| `add_drop_threshold` | 加仓跌幅阈值（%） | 2.0 - 5.0 | 值越小，加仓越频繁 |
| `take_profit_threshold` | 止盈涨幅阈值（%） | 1.5 - 4.0 | 值越小，止盈越快 |
| `max_add_positions` | 最大加仓次数 | 3 - 6 | 控制最大风险暴露 |
| `initial_position_pct` | 初始建仓比例（%） | 4 - 10 | 控制初始资金投入 |

## 🔧 配置示例

### 最简配置（只改两个核心参数）

```yaml
symbols:
  - code: "sh.513120"
    name: "港股创新药ETF"
    enabled: true
    add_drop_threshold: 3.0  # 下跌3%加仓
    take_profit_threshold: 2.0  # 盈利2%止盈
```

### 完整配置（所有参数都个性化）

```yaml
symbols:
  - code: "sh.513120"
    name: "港股创新药ETF"
    enabled: true
    add_drop_threshold: 3.0
    take_profit_threshold: 2.0
    max_add_positions: 4
    initial_position_pct: 6
```

### 混合配置（部分个性化，部分用全局默认）

```yaml
symbols:
  - code: "sh.513120"
    name: "港股创新药ETF"
    enabled: true
    add_drop_threshold: 3.0  # 个性化
    # take_profit_threshold 不设置，使用全局默认值
    
strategy:
  take_profit_threshold: 2.5  # 全局默认值
```

## 💡 实用技巧

### 技巧 1：渐进式调整

```
第1周：使用默认配置观察
第2周：小幅调整 ±0.5%
第3周：根据效果继续优化
每月：回顾一次，持续改进
```

### 技巧 2：记录对比

创建参数调整记录表：

| 日期 | ETF | 加仓阈值 | 止盈阈值 | 交易次数 | 总收益 |
|------|-----|----------|----------|----------|--------|
| 04-01 | 513120 | 3.0% | 2.0% | 5 | +3.2% |
| 04-15 | 513120 | 2.5% | 1.5% | 8 | +4.1% |

### 技巧 3：避免常见错误

❌ **错误**：所有 ETF 用相同参数  
✅ **正确**：根据每个 ETF 的特性分别设置

❌ **错误**：参数设置后不再调整  
✅ **正确**：定期回顾和优化

❌ **错误**：一次性大幅调整  
✅ **正确**：小步快跑，逐步优化

## 🔍 优先级规则

```
如果 symbol 配置了参数 → 使用该值
否则 → 使用 strategy 中的全局默认值
```

**示例**：
```yaml
symbols:
  - code: "sh.513120"
    add_drop_threshold: 3.0  # ← 使用这个值（3.0%）
    
strategy:
  add_drop_threshold: 3.5  # ← 全局默认值（仅当 symbol 未配置时使用）
```

## 📊 效果评估指标

观察以下指标来判断参数是否合适：

1. **交易频率**：每周 2-5 次为宜
2. **胜率**：盈利交易占比 > 60%
3. **平均持仓时间**：1-7 天
4. **最大回撤**：< 10%
5. **总收益率**：月化 > 3%

如果指标不理想，适当调整参数。

## ⚠️ 注意事项

1. **不要过度优化**：参数太多会增加复杂度
2. **考虑交易成本**：频繁交易会增加手续费
3. **预留充足资金**：确保有足够的资金用于加仓
4. **定期备份配置**：保存历史配置文件
5. **实盘前充分测试**：先用模拟或小资金测试

## 🆘 常见问题

**Q: 如何恢复默认配置？**  
A: 删除 symbol 下的个性化参数，系统会自动使用全局默认值。

**Q: 可以只配置一个参数吗？**  
A: 可以！只配置需要的参数，其他参数会使用全局默认值。

**Q: 修改配置后需要重启系统吗？**  
A: 是的，修改 `config.yaml` 后需要重启 `main.py`。

**Q: 如何知道当前使用的是哪个配置？**  
A: 查看启动日志，`✓` 表示个性化配置，`○` 表示全局默认配置。
