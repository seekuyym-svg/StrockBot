# 飞书通知功能使用指南

## 🎯 功能概述

系统现已支持通过**飞书机器人**实时推送交易信号通知，当触发买入、卖出、加仓或止损信号时，会立即将详细信息推送到您的飞书群聊。

## 📋 配置步骤

### 第一步：获取飞书机器人 Webhook URL

1. **打开飞书**，进入您想接收通知的群聊
2. 点击右上角 **"..."** → **"群机器人"** → **"添加机器人"**
3. 选择 **"自定义机器人"**，点击 **"添加"**
4. 设置机器人名称（如：**ETF交易助手**）
5. （可选）上传机器人头像
6. 点击 **"完成"**
7. **复制生成的 Webhook 地址**（格式类似：`https://open.feishu.cn/open-apis/bot/v2/hook/xxxxx`）

⚠️ **重要提示：**
- 妥善保管 Webhook URL，不要泄露给他人
- 一个机器人可以添加到多个群聊
- 建议在专门的监控群中使用

### 第二步：配置 config.yaml

编辑 `config.yaml` 文件，在 `notification` 部分配置：

```yaml
# 通知配置
notification:
  # 飞书机器人通知
  feishu:
    # 是否启用飞书通知
    enabled: true
    # 飞书机器人 Webhook URL（替换为您的实际URL）
    webhook_url: "https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
    # 通知类型：BUY/SELL/ADD/STOP
    notify_signals: ["BUY", "SELL", "ADD", "STOP"]
```

**配置项说明：**
- `enabled`: 是否启用飞书通知（true/false）
- `webhook_url`: 您的飞书机器人 Webhook URL
- `notify_signals`: 需要通知的信号类型列表
  - `BUY`: 买入信号
  - `SELL`: 卖出信号
  - `ADD`: 加仓信号
  - `STOP`: 止损信号
  - `WAIT`: 观望信号（默认不通知，避免刷屏）

### 第三步：重启系统

```bash
python main.py
```

启动后，您会看到日志输出：

```
✅ 飞书通知已启用，将通知以下信号类型: ['BUY', 'SELL', 'ADD', 'STOP']
```

## ⚙️ 高级配置

### 选择性通知

如果您只想接收某些类型的信号，可以修改 `notify_signals`：

**只接收买卖信号：**
```yaml
notification:
  feishu:
    enabled: true
    webhook_url: "your_webhook_url"
    notify_signals: ["BUY", "SELL"]  # 仅买卖
```

**接收所有信号（包括加仓）：**
```yaml
notification:
  feishu:
    enabled: true
    webhook_url: "your_webhook_url"
    notify_signals: ["BUY", "SELL", "ADD", "STOP"]
```

**临时关闭通知：**
```yaml
notification:
  feishu:
    enabled: false  # 禁用但不删除配置
```

### 多群通知

如果需要同时通知多个群，可以创建多个飞书机器人，然后修改代码支持多个Webhook（当前版本仅支持单个）。

## 🔍 故障排查

### 问题1: 收不到通知

**可能原因：**
1. `enabled` 设置为 `false`
2. `webhook_url` 配置错误或为空
3. 网络连接问题
4. 飞书机器人被移除或禁用

**解决方法：**
```bash
# 1. 检查配置
cat config.yaml | grep -A 5 feishu

# 2. 运行测试脚本
python test_feishu_notification.py

# 3. 查看日志
# 启动时应该看到：✅ 飞书通知已启用

# 4. 检查飞书群聊
# 确认机器人仍在群中且未被禁用
```

### 问题2: 通知发送失败

**日志显示：**
```
❌ 飞书通知发送失败，HTTP状态码: 403
```

**可能原因：**
- Webhook URL 无效或已过期
- 飞书机器人权限不足

**解决方法：**
1. 重新生成 Webhook URL
2. 确认机器人有发送消息权限
3. 检查群聊设置

### 问题3: 通知延迟

**可能原因：**
- 网络波动
- 飞书服务器响应慢

**解决方法：**
- 检查网络连接
- 稍后重试
- 联系飞书技术支持

### 问题4: 消息格式异常

**现象：** 收到的消息格式不正确

**解决方法：**
- 更新到最新版本
- 检查飞书客户端版本
- 清除飞书缓存

## 💡 最佳实践

### 1. 专用监控群

建议创建一个专门的群聊用于接收交易通知：
- 群名：**ETF交易监控**
- 成员：仅您自己或核心团队成员
- 目的：避免打扰其他群成员

### 2. 通知频率控制

为避免频繁通知造成干扰：
- 默认不通知 WAIT 信号
- 可根据需要调整 `notify_signals`
- 考虑设置免打扰时段

### 3. 备份配置

定期备份 `config.yaml`，特别是 Webhook URL：
```bash
cp config.yaml config.yaml.backup
```

### 4. 安全注意事项

- ⚠️ **不要公开分享 Webhook URL**
- ⚠️ 定期更换 Webhook（飞书支持重新生成）
- ⚠️ 如发现异常消息，立即禁用并重新生成

### 5. 配合其他工具

可以将飞书通知与其他工具结合：
- **飞书日历**：记录重要交易时间
- **飞书文档**：整理交易日志
